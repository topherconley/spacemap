Tuning Spacemap

Chris Conley

March 11, 2016

Tuning Strategy for Managing Computational Constraints

The three tuning parameters in Spacemap provides a rich framework for discovering conditional dependencies between variables in high dimensional biological assays; however, exploring the tuning space can be very computationally demanding. In this tutorial we provided a strategy for finding the neighborhood of good tuning parameters given computational limits. The strategy finds suitiable neighborhoods of each tuning parameter by:

  1. Tuning Space on the full data (X,Y) and Y alone to find a neighborhood of the first tuning parameter, denoted as slasso.
  2. Tuning Spacemap with the neigbhorhood of slasso from step 1, and exploring a broad range of values for rlasso. while holding rgroup=0.
  3. Tuning Spacemap with local neighborhoods on slasso and rlasso discovered from steps 1 & 2, while exploring a broad range of values for rgroup > 0.

We will illustrate the details of this strategy with an example from a simulation. The known truth of the simulation affords an evaluation addressing whether this strategy leads to reasonably tuned parameter selection.

Find a neighborhood for slasso

Tune the slasso parameter by fitting the Space model to either the response Y or jointly (X,Y) over a one-dimensional tuning grid. In the current implementation, the tuning parameter scales with the sample size. To find the right tuning scale, use a result from Meihshausen and Buhlmann (2006) which applies when when all Y have been standardized to have unit variance. The value of alpha is meant to control the false discovery rate.

lam1start <- function(n, q, alpha) { 
  sqrt(n) * qnorm(1 - (alpha/ (2*q^2)))
}
lam0 <- lam1start(n = floor(N - N*.10), q = Q, alpha = 0.1)
lam0
## [1] 53.95659

We want to select a neighborhood using this initial value, where lam0 serves as an approximate lower bound of the neighborhood. We use it as a lower bound because we prefer to start with parsimonious models. Take the initial neighborhood to be [-10% of lam0, 190% of lam0] as a starting point.

#neighborhood parameter about lam0. 
eps.10 <- 0.1
#initial grid size. 
ngrid <- 50
#grid
tsp <- expand.grid(slasso = seq(lam0*(1 - eps.10), lam0*(1 + (1 - eps.10)), length = ngrid))
summary(tsp)

Since cross-validation is expensive, we can do an initial fit with all samples on (X,Y) to get a sense of the model complexity or the models degrees of freedom (number of edges) for the chosen neighborhood.

#convergence tolerance
tol <- 1e-6
#max. numbr of iterations for coordinate descent optimation
cd_iter <- 1e7
#number of iterations for estimating diagonal of concentration matrix.
iter <- 3
#ridge parameter for space (for elastic net if desired)
sridge <- 0
#ridge parameter for refit
refitRidge = 0.0
#fit the model to all samples 
dfr <- spacemap::initFit(dat = dat, method = "space", 
                         tuneGrid = tsp, 
                         sridge = 0, tol = tol, cd_iter = cd_iter, iter = iter, 
                         Yindex = dat$Yindex, Xindex = dat$Xindex)

While there are a number of metrics recorded from initFit, we care primarily about the number of Y--Y edges reported as dfParCor and X->Y reported as dfGamma. The total number of edges is reported as dfTotal

boxplot(dfr[,c("dfTotal", "dfParCor", "dfGamma")], pch = 19)

One ought to determine if the degrees of freedom is too high (tuning parameter too low) from factors like how much power from the sample size is afforded, what level FDR is tolerated, and how sparse the network is anticipated to be. Making that determination is not easy, but realistic estimates can dramatically improve computational times. Since we have the true cardinality of the edge set available to us, we can compare the true model comlexity to the complexity of models generated under the initial tuning neighborhood.

#true X->Y edges
nonZeroWhole(sim1$trueParCor$xy, tol)
## [1] 54
#true Y--Y edges
nonZeroUpper(sim1$trueParCor$yy, tol)
## [1] 156

Notice that the initial neighborhood generates fitted model complexities that bounds the true model complexity; this gives confidence that the neighborhood selection procedure is suitable for this example. The more degrees of freedom a model has, the longer it typically takes for model convergence. Selecting a tuning grid that generates models with unreasonably high degrees of freedom will make the cross-validation step take unnecessarily long. This initial fitting procedure helps eliminate unnecessary computation. Since the maximum degrees of freedom under this neighborhood is not extreme, proceed to create cross validation training and test sets.

The power is somewhat limited but the false discovery rate is controlled to reasonable levels with the vote procedure producing the lowest FDR with some cost to the power. The vote procedure is preferred to the 1-sd rule since it renders comparable power with lower FDR. The X->Y power and FDR are smaller than the Y--Y power. Inspect the CV-vote-error curve and confirm that the curve is well-behaved. Find a neighborhood for rlasso

Set up a suitable tuning grid with slasso in a neighborhood based on the previous tuning. We expect the rlasso parameter to be on approximately the same scale as slasso. Just to check, we fit with rlasso = slasso to see what the number of X->Y edges are returned.

lam1 <- cvSpaceYY$tuneGrid[1]
eps.15 <- 0.15
tg0 <- expand.grid(slasso = seq(lam1*(1 - eps.15), lam1*(1 + eps.15), length = 10), 
                    rlasso = lam1, 
                    rgroup = 0)
dfr <- spacemap::initFit(dat = dat, method = "spacemap", 
                         tuneGrid = tg0, 
                         sridge = sridge, tol = tol, cd_iter = cd_iter, iter = iter)
boxplot(dfr[,c("dfTotal", "dfParCor", "dfGamma")])

The penalty is too high, with very few X->Y edges being considered. Vary rlasso downward as a sequence from (85% optimized slasso, 15% optimized slasso). Vary slasso as +/- 15% of the optimized value from previous tunings. Space the grid points on a squared scale. The squared scale grid generates more parsimonious models relative to large models. Essentially, it is computationally wasteful to fit large models until we know these models are better than parsimonious models.

eps.15 <- 0.15
lamg0Lower <- lam1*eps.15
lamg0Upper <- lam1*(1 - eps.15)
tg0 <- expand.grid(slasso = sqrt(seq((lam1*(1 - eps.15))^2, (lam1*(1 + eps.15))^2, length = 5)), 
                    rlasso = sqrt(seq(lamg0Upper^2, lamg0Lower^2, length  = 10)), 
                    rgroup = 0)

Assess whether the tuning grid generates parsimonious models as well as a few large models. The rlasso paramter will mostly control the size of dfGamma and have limited influence of dfParCor.

dfr <- spacemap::initFit(dat = dat, method = "spacemap", 
                         tuneGrid = tg0, 
                         sridge = sridge, tol = tol, cd_iter = cd_iter, iter = iter)
boxplot(dfr[,c("dfTotal", "dfParCor", "dfGamma")], pch = 19)

Find a neighborhood for rgroup

Construct a tuning grid using the previous tunings information. Use the same slasso tuning grid from the tuning of Spacemap (Group = 0). If there is a non-zero group penalty, the CV-error selected rlasso penalty is generally smaller from the (Group = 0) tuning. We set the rlasso penalty grid to be between 50% and 110% of the selected value anticipating some decrease in global X->Y penalty. Both slasso and rlasso are placed on the squared scale. The rgroup penalty is bounded between zero and the rlasso selected from (Group = 0) tuning. The rgroup penalty is on the log scale for several reasons; (i) we already have a well-tuned slasso and rlasso penalties maintaing global sparisty; (ii) this final tuning is more subtle and is looking to find any master predictors that influence multiple responses. The size of the tuning grid depends on computational resources available. We recommend starting small and iteratively narrowing down the neighborhood.

eps.50 <- 0.5
eps.10 <- 0.1
tmap <- expand.grid(slasso = sort(unique(tg02$slasso)), 
                    rlasso = sqrt(seq((lam2.2*(1 - eps.50))^2, (lam2.2*(1 + eps.10))^2, length = 5)), 
                    rgroup = exp(seq(log(1.5), log(lam2.2), length = 5)))

Refine Spacemap Grid

Well-designed tuning grids inform future tuning refinements. We narrow the range of each tuning parameter to be within some neighborhood of the current optimized value: +/- 7.5% of slasso; +/- 15% of rlasso; and +/- 40% of rgroup. The range of the parameters is inversely related to the level of confidence we have in the current optimized value.

eps.075 <- 0.075
lam2.3 <- cvSpmap$tuneGrid$rlasso

Evaluating the tuning cost and benefits.

Disciplined tuning of Spacemap parameters yields real improvements in power at comparablely low FDR thresholds for this hub network simulation, where some hubs are predictor variables. We have provided a reasonable tuning framework that balances the trade-off of model accuracy and computational speed. Testing the robustness of this tuning framework needs to be explored with respect to changes in sample size, network topology, and other factors like signal to noise ratio and distributional assumptions.
